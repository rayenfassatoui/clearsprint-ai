## Your Core Principles

### 1. Always Be Organized
- Think systematically before you code.
- Plan the architecture, data flow, and component hierarchy.
- Keep files, functions, and logic clearly structured.
- Use consistent naming conventions and folder organization.

### 2. Design Modern & Elegant
- Every UI must be **clean, refined, and visually balanced**.
- Use **ample spacing**, proper typography hierarchy, and harmonious colors.
- Never leave empty spots — every pixel should serve a purpose.
- Ensure **breathing room** between elements; cards and sections should feel spacious.
- Make layouts **unconstrained** — let content flow naturally without rigid limits.
- Always prioritize **visual harmony** and **perfect balance**.

### 3. Build Beautiful UX with Micro-Interactions & Animations
- Add **subtle animations** that enhance, not distract (fade-ins, hovers, transitions).
- Include **micro-interactions** for buttons, links, and inputs (scale, color shifts, smooth transitions).
- Use CSS animations sparingly — every animation should feel intentional and smooth.
- Ensure animations use `transition` utilities and CSS for performance.

### 4. Write Mobile-First, Tailwind-First Code
- Start with **mobile design**, then progressively enhance for larger screens.
- Use **Tailwind CSS utility classes** exclusively — no custom CSS unless absolutely necessary.
- Keep styling **responsive** with breakpoints: `sm:`, `md:`, `lg:`, `xl:`.
- Use Tailwind's **dark mode** support; ensure the design works in all contexts.

### 5. Code Quality & Best Practices
- Write **type-safe TypeScript** — strict typing everywhere.
- Use **server components** by default; only client components when needed (`"use client"`).
- Follow **DRY** and **KISS** principles — no duplication, keep logic simple.
- Validate data at **API** and **UI** layers.
- Handle errors gracefully with clear, user-friendly messages.
- Write **self-explanatory code** — minimal comments, clear intent.

### 6. Component Architecture
- Use **Shadcn/UI components** as the foundation for all UI.
- Extend or compose components only when Shadcn doesn't cover your needs.
- Keep components **reusable, composable, and accessible** (WCAG standards).
- Prop-driven design — make components flexible and configurable.

## Design Standards

- **Spacing**: Use consistent gaps (`gap-4`, `gap-6`, `gap-8`, etc.) throughout.
- **Typography**: Clear hierarchy with size, weight, and color differentiation.
- **Colors**: Stick to the theme palette; use opacity for depth (`/50`, `/10`, etc.).
- **Borders & Shadows**: Subtle, refined touches — avoid harsh contrasts.
- **Hover States**: Always include smooth transitions and visual feedback.
- **Accessibility**: Semantic HTML, proper ARIA labels, keyboard navigation, sufficient contrast.
